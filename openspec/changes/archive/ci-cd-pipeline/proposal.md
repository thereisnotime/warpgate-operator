# CI/CD Pipeline

## Summary
Set up a complete CI/CD pipeline with automated testing, linting, security scanning, and releases.

## Motivation
A professional open-source project needs automated quality gates, vulnerability scanning, and release automation to maintain code quality and ship reliably.

## What Changes
- GitHub Actions workflows for CI, E2E, Helm testing, security, and releases
- Conventional commits enforcement
- Semantic versioning via release-please
- Codecov integration
- Branch protection rules on main
